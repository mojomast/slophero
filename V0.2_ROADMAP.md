## üéØ DESIGN PHILOSOPHY

**v0.1 was about:** Can we build it? Does it work? Can we ship it?  
**v0.2 is about:** Does it feel ALIVE? Do players understand it? Do they want MORE?

### The Three Pillars of v0.2

1. **TRANSPARENCY** - Players know what's happening and why
   - No hidden mechanics that feel like RNG
   - Clear previews of consequences
   - Visible progression systems

2. **CONSEQUENCE** - Choices echo through time
   - Early decisions affect late game
   - Celebrity relationships persist
   - Crew remembers your style

3. **EMERGENCE** - Depth without complexity
   - Simple rules create complex situations
   - Player-discovered strategies
   - "I didn't know you could do that" moments

## üìã PRIORITY 2: CLEAR CAUSE ‚Üí EFFECT

### Problem Statement (v0.1)

**Player confusion:**
- "I chose that and my Heat went up? Why?"
- "What does this button even do?"
- "Is this good or bad? I can't tell."
- "It feels like random outcomes."

**The invisible mechanics problem:**
Good game design often hides complexity. But ROO GANG's complexity isn't elegant - it's just confusing. Players can't strategize if they don't understand the game.

**Design principle:** 
> "A player should never be surprised by a mechanic, only by an outcome."

### The Big Idea

Every choice should communicate:
1. **What will happen** (preview)
2. **Why it happened** (confirmation)
3. **What it unlocked** (consequence chains)

Think: Slay the Spire's combat preview, FTL's clear risk/reward, Into the Breach's perfect information.

#### **Risk Levels** üé≤

**The Four Tiers:**

1. **üü¢ SAFE** - Guaranteed outcome, no surprises

   - Example: "Use backup server" ‚Üí +Tokens, -Energy

   - Reward: Medium, proportional to cost

3. **üü† RISKY** - High variance
   - Example: "Try experimental route" ‚Üí 60% success (huge gain) / 40% fail (Heat spike)
   - Use case: When desperate or confident
   - Reward: High if it works, punishing if it doesn't

   - Example: "Let EvilFear handle it" ‚Üí Completely unpredictable
   - Use case: When you don't care anymore or want maximum chaos

## üìù TECHNICAL IMPLEMENTATION GUIDE

### Architecture Overview

**v0.2 adds three major systems:**

1. **Progression System** - Tracks perk tiers, usage counts, unlocks
2. **Memory System** - Stores choice history, relationship tags, event chains
3. **Preview System** - Parses Ink tags, displays tooltips, calculates outcomes

### New Data Structures

```typescript
// ============================================
// PROGRESSION SYSTEM
// ============================================

interface PerkState {
  id: string;                    // 'hubris-breather'
  tier: 1 | 2 | 3 | 4;           // Current tier (4 = secret mastery)
  usageCount: number;            // Total times used
  lastUsedCycle: number;         // For cooldown tracking
  cooldownRemaining: number;     // Cycles until available
  unlockedAt: number;            // Cycle when tier unlocked
  evolutionHistory: TierUnlock[]; // Animation purposes
}

interface TierUnlock {
  tier: number;
  cycle: number;
  witnessedByPlayer: boolean;    // Did they see the animation?
}

interface PerkCombo {
  perks: string[];               // ['breather', 'rtc']
  name: string;                  // 'Calm Before Storm'
  effect: StatDelta;             // Bonus stats
  triggeredCount: number;        // For achievements
}

// ============================================
// MEMORY SYSTEM
// ============================================

interface ChoiceMemory {
  id: string;                    // Unique identifier
  cycle: number;                 // When it happened
  operation: OperationType;      // Which op
  choiceText: string;            // What they clicked
  tags: string[];                // ['helped-elon', 'burned-favor']
  consequences: string[];        // What unlocked
  statDelta: StatDelta;          // Net effect
}

interface RelationshipState {
  npcId: string;                 // 'elon-musk'
  level: number;                 // -100 to +100
  interactions: number;          // How many times met
  lastInteraction: number;       // Cycle number
  tags: string[];                // ['helped', 'ignored', 'trolled']
  unlockedEvents: string[];      // Future encounters available
}

interface EncounterChain {
  id: string;                    // 'vc-spiral'
  name: string;                  // 'The VC Spiral'
  currentStep: number;           // Which part of chain
  totalSteps: number;            // How many parts total
  triggeredCycle: number;        // When it started
  choicesMade: ChainChoice[];    // History
  isActive: boolean;             // Still ongoing?
  canBranch: boolean;            // Multiple paths?
}

interface ChainChoice {
  step: number;
  cycle: number;
  choice: string;
  tags: string[];
}

// ============================================
// PREVIEW SYSTEM
// ============================================

interface ChoicePreview {
  choiceText: string;
  riskLevel: 'safe' | 'balanced' | 'risky' | 'yolo';
  immediateEffects: StatDelta;
  risks: RiskFactor[];
  unlocks: string[];
  requirements: string[];        // Tags needed to see this choice
  probability: number | 'hidden'; // Success chance or ???
}

interface RiskFactor {
  description: string;           // "roo5 might not help next time"
  probability: number;           // 0-100 or null if unknown
  severity: 'low' | 'medium' | 'high' | 'catastrophic';
}

interface StatDelta {
  energy?: number;
  sanity?: number;
  heat?: number;
  tokens?: number;
  hype?: number;
  cred?: number;
  trust?: number;                // Hidden stat
  chaos?: number;                // New v0.2 stat
}

// ============================================
// CHAOS METER
// ============================================

interface ChaosState {
  level: number;                 // 0-100
  modifiers: ChaosModifier[];
  encounterMultiplier: number;   // Affects spawn rate
  lastSpike: number;             // Cycle of last major event
}

interface ChaosModifier {
  id: string;                    // 'elon-tweeted'
  name: string;                  // 'Elon Tweeted About You'
  effect: number;                // +10 chaos
  expiresAt: number;             // Cycle when it wears off
  icon: string;                  // For UI display
}
```
